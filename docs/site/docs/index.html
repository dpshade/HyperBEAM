DOCUMENTATION

This guide will walk you through the steps to set up and run hyperBEAM and the local computational unit (CU), explaining their integration, configuration options, and optimization strategies. You'll also find troubleshooting tips, FAQs, and additional resources for advanced usage.

INFO: This documentation currently supports Ubuntu 22.04 only. Support for macOS will be added in a future update.

Prerequisites

System Requirements

  • Operating System: Ubuntu 22.04
  • CPU: 4+ cores recommended
  • RAM: 8GB minimum, 16GB recommended

Required Software

The following software packages are required:

  • build-essential: Contains basic build tools including gcc/g++ compilers
  • cmake: Cross-platform build system
  • git: Version control system
  • pkg-config: Helper tool for compiling applications and libraries
  • ncurses-dev: Development libraries for terminal-based interfaces
  • libssl-dev: Development libraries for SSL (Secure Sockets Layer)
  • sudo: Allows running programs with security privileges of another user
  • curl: Command line tool for transferring data with URL syntax
  • ca-certificates: Common CA certificates for SSL applications

Install all dependencies with the following command:

sudo apt-get update && sudo apt-get install -y --no-install-recommends \
    build-essential cmake git pkg-config ncurses-dev libssl-dev sudo curl ca-certificates

Build and Install Erlang/OTP

git clone https://github.com/erlang/otp.git
cd otp
git checkout maint-27
./configure --without-wx --without-debugger --without-observer --without-et
make -j$(nproc)
sudo make install
cd ..
rm -rf otp

Build and Install Rebar3

git clone https://github.com/erlang/rebar3.git
cd rebar3
./bootstrap
sudo mv rebar3 /usr/local/bin/
cd ..
rm -rf rebar3

Install Rust and Cargo

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y --default-toolchain stable

Install Node.js (includes npm and npx)

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - && \
sudo apt-get install -y nodejs && \
node -v && npm -v

HyperBeam Setup

What is HyperBeam?

HyperBeam is a client implementation of the AO-Core protocol, written in Erlang. It serves as the 'node' software for the decentralized operating system that AO enables, abstracting hardware provisioning and details from the execution of individual programs.

HyperBEAM node operators can offer their machine's services to others in the network by executing various devices and charging users for computation as needed.

Prerequisites

Before installing HyperBeam, ensure you have the following:

  • The Erlang runtime (OTP 27)
  • Rebar3
  • Git

Installation Steps

Clone the HyperBEAM Repository

git clone https://github.com/permaweb/HyperBEAM.git
cd HyperBEAM

Compile HyperBEAM

rebar3 compile

Configuration

HyperBeam can be configured using a ~meta@1.0 device, initialized via command line arguments when starting the node.

Basic Configuration

rebar3 shell --eval "hb:start_mainnet(#{ port => 9001, key_location => 'path/to/my/wallet.key' })."

Using a Configuration File

Node operators can also configure using a flat@1.0 encoded settings file. Example format:

port: 1337
cache-lookup-hueristics: true

Local CU Setup

What is Local CU?

The ao Compute Unit (CU) is a spec-compliant implementation built with NodeJS. It serves as the computational processing component in the ao ecosystem, handling WASM execution and state management.

Prerequisites

  • Node.js
  • Git

Installation Steps

1. Clone the Repository

git clone https://github.com/permaweb/ao.git
cd ao/servers/cu

2. Install Dependencies

npm i

3. Generate a Wallet (if needed)

If you don't already have an Arweave wallet, you can generate one:

npx --yes @permaweb/wallet > wallet.json

4. Configure Environment

Create a .env file with the following minimal configuration:

UNIT_MODE=hbu
HB_URL=http://localhost:10000
PORT=6363
WALLET_FILE=./wallet.json
NODE_CONFIG_ENV="development"

5. Start the Compute Unit

npm start

For development with hot-reloading, you can use:

npm run dev

Key Configuration Options

The CU supports numerous environment variables for advanced configuration. Here are some important ones:

  • WALLET_FILE: Path to your Arweave wallet JSON file
  • PORT: The port on which the CU server will listen (default: 6363)
  • UNIT_MODE: Set to "hbu" for hyperBEAM mode
  • HB_URL: URL of your hyperBEAM instance
  • PROCESS_WASM_MEMORY_MAX_LIMIT: Maximum memory limit for processes (default: 1GB)
  • DEFAULT_LOG_LEVEL: Logging level (error, warn, info, http, verbose, debug, silly)

Verification

To verify that your CU is running correctly, you can check:

curl http://localhost:6363/health

This should return a status indicator showing the CU is operational.

Next Steps

Using HyperBeam and CU Together

With both HyperBeam and the Local CU running, you can now interact with the system through HTTP endpoints.

Interacting with HyperBeam via HTTP

HyperBeam exposes HTTP endpoints that you can use to send messages and interact with its devices. Here are some examples:

Get Node Information

curl http://localhost:10000/~meta@1.0/info

Check Node Status

curl http://localhost:10000/status

Using AOS CLI for Testing

The aos CLI provides a convenient way to test your HyperBeam and CU setup:

1. Install the AOS CLI

npm i -g https://arweave.net/XCeKQlNLulA6IZCHZQsZcr3wWomXLJrvowJH05fV_m4

2. Test HyperBeam Connection

aos hb_test --mainnet http://localhost:10000

Verifying End-to-End Functionality

To ensure both services are working together properly:

  1. Check that HyperBeam can communicate with your Local CU by examining logs
  2. Send a test message through HyperBeam that requires CU execution
  3. Verify the message is processed by both components
